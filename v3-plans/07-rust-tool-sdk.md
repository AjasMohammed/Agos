# Plan 07 — Rust Tool SDK (`agentos-sdk`)

## Goal

Build the `agentos-sdk` crate — a proc-macro-powered ergonomic API that lets tool developers write a tool in Rust as a simple annotated function, with the SDK generating all the boilerplate: manifest TOML, WASI stdin/stdout wiring, serialisation, and the `AgentTool` trait implementation.

---

## The Problem Without an SDK

Today, writing a WASM tool in Rust requires:

1. Manually writing a `ToolManifest` TOML
2. Manually reading from `stdin`, parsing JSON
3. Manually writing to `$AGENTOS_OUTPUT_FILE`
4. Knowing the WASI `wasm32-wasip1` target details
5. Writing boilerplate `main()` and error handling

The SDK collapses all of this into a single annotated function.

---

## What the SDK Provides

```rust
use agentos_sdk::tool;

#[tool(
    name = "web-search",
    version = "1.0.0",
    description = "Searches the web and returns structured results.",
    permissions = ["network.outbound:x"],  // ":x" = Execute permission (":r" = Read, ":w" = Write)
    network = true,  // Global toggle — must be true for any network permissions to apply; permissions further restrict allowed operations
    max_memory_mb = 128,
    max_cpu_ms = 15000,
)]
async fn web_search(intent: WebSearchIntent) -> ToolResult<SearchResults> {
    // Your logic here. HTTP, file I/O, whatever you need.
    let results = make_http_call(&intent.query).await?;
    Ok(SearchResults { items: parse(results) })
}
```

The `#[tool]` macro generates:

1. A `main()` function that reads payload from stdin and writes the result to `$AGENTOS_OUTPUT_FILE`
2. A `web-search.toml` manifest file (emitted as a build artifact)
3. Input/output serde validation
4. Standardised error response on failure

## Proc-Macro: `#[tool(...)]`

The macro is applied to an `async fn` and generates a complete binary entry point.

### Input

```rust
#[tool(
    name = "sentiment-classifier",
    version = "1.0.0",
    description = "Classifies text sentiment as positive, negative, or neutral.",
    permissions = [],
    network = false,
    max_memory_mb = 64,
    max_cpu_ms = 2000,
)]
async fn sentiment_classifier(intent: SentimentRequest) -> ToolResult<SentimentResponse> {
    let score = classify(&intent.text);
    Ok(SentimentResponse { sentiment: score.label, confidence: score.confidence })
}
```

### Generated Code

```rust
// Generated by #[tool(...)]
#[tokio::main]
async fn main() {
    // Read JSON payload from stdin (the AgentOS contract)
    let intent: SentimentRequest = match agentos_sdk::io::read_input() {
        Ok(v) => v,
        Err(e) => {
            eprintln!("Failed to read SentimentRequest from stdin: {}", e);
            std::process::exit(1);
        }
    };

    // Execute the user's function
    let result: ToolResult<SentimentResponse> = sentiment_classifier(intent).await;

    // Write result (or error) to AGENTOS_OUTPUT_FILE
    match result {
        Ok(output) => {
            if let Err(e) = agentos_sdk::io::write_output(&output) {
                eprintln!("Failed to write output: {}", e);
                std::process::exit(1);
            }
        }
        Err(err) => {
            if let Err(e) = agentos_sdk::io::write_error(&err) {
                eprintln!("Failed to write error: {}", e);
            }
            std::process::exit(1);
        }
    }
}
```

## ToolContext

Some tools need access to secrets (without seeing raw values) or the kernel-provided trace ID. `ToolContext` is automatically passed by adding it as a parameter:

```rust
#[tool(name = "slack-notifier", version = "1.0.0", ...)]
async fn slack_notifier(mut ctx: ToolContext, intent: SlackIntent) -> ToolResult<()> {
    // vault-resolved secret — removed from context after first read (zero-after-read)
    let token = ctx.secret("SLACK_TOKEN")?;  // Synchronous — secrets are pre-loaded by the kernel

    ctx.log_debug("Posting message to Slack...");

    post_to_slack(&token, &intent.channel, &intent.message).await?;
    Ok(())
}
```

```rust
use secrecy::{ExposeSecret, SecretString};

pub struct ToolContext {
    /// Resolved secrets from the vault — stored as SecretString to prevent
    /// accidental logging. Secrets are removed after first read (zero-after-read).
    secret_env: HashMap<String, SecretString>,

    /// Trace ID for correlation with the kernel audit log.
    pub trace_id: String,

    /// Task ID of the calling agent task.
    pub task_id: String,
}

impl ToolContext {
    /// Get a secret the kernel injected via env var.
    /// The secret is **removed** from the map after first read (zero-after-read)
    /// to minimise the window where it exists in memory.
    pub fn secret(&mut self, name: &str) -> Result<String, ToolError> {
        self.secret_env.remove(name)
            .map(|s| s.expose_secret().to_string())
            .ok_or(ToolError::SecretNotFound(name.to_string()))
    }

    pub fn log_debug(&self, msg: &str) { eprintln!("[DEBUG] {}", msg); }
    pub fn log_info(&self,  msg: &str) { eprintln!("[INFO]  {}", msg); }
    pub fn log_warn(&self,  msg: &str) { eprintln!("[WARN]  {}", msg); }
    pub fn log_error(&self, msg: &str) { eprintln!("[ERROR] {}", msg); }
}
```

> Secrets are injected by the kernel as environment variables prefixed with `AGENTOS_SECRET_`. E.g. `SLACK_TOKEN` → `AGENTOS_SECRET_SLACK_TOKEN`. On `ToolContext` construction, env vars are read into `SecretString` values (from the `secrecy` crate) and the env vars are cleared. `ToolContext::secret()` then uses `remove()` to return the value exactly once — subsequent calls for the same key return `SecretNotFound`.

## I/O Helpers

```rust
// agentos-sdk/src/io.rs

/// Read and deserialise the JSON payload from stdin.
pub fn read_input<T: DeserializeOwned>() -> Result<T, ToolError> {
    let mut buf = String::new();
    std::io::stdin().read_to_string(&mut buf)?;
    serde_json::from_str(&buf).map_err(ToolError::Deserialisation)
}

/// Write a JSON result to the AGENTOS_OUTPUT_FILE path.
pub fn write_output<T: Serialize>(value: &T) -> Result<(), ToolError> {
    let path = output_file_path()?;
    let json = serde_json::to_string_pretty(value)?;
    std::fs::write(&path, json)?;
    Ok(())
}

/// Write a structured error to the output file, falling back to stderr.
pub fn write_error(err: &ToolError) -> Result<(), ToolError> {
    let error_payload = serde_json::json!({
        "error": {
            "code": err.code(),
            "message": err.to_string()
        }
    });
    if let Err(e) = write_output(&error_payload) {
        eprintln!("write_error fallback: {}", serde_json::to_string(&error_payload).unwrap_or_else(|_| err.to_string()));
        return Err(e);
    }
    Ok(())
}

/// Get the unique output file path injected by the kernel.
pub fn output_file_path() -> Result<PathBuf, ToolError> {
    std::env::var("AGENTOS_OUTPUT_FILE")
        .map(PathBuf::from)
        .map_err(|_| ToolError::MissingOutputFile)
}
```

## Error Type

```rust
#[derive(Debug, thiserror::Error)]
pub enum ToolError {
    #[error("IO error: {0}")]
    Io(#[from] std::io::Error),

    #[error("Deserialisation failed: {0}")]
    Deserialisation(serde_json::Error),

    #[error("Serialisation failed: {0}")]
    Serialisation(serde_json::Error),

    #[error("Secret '{0}' not found — ensure it is set via agentctl secret set")]
    SecretNotFound(String),

    #[error("AGENTOS_OUTPUT_FILE not set — is this running inside AgentOS?")]
    MissingOutputFile,

    #[error("Tool logic error: {0}")]
    Logic(String),

    #[error("HTTP error: {0}")]
    Http(String),
}

pub type ToolResult<T> = Result<T, ToolError>;
```

## Developer Workflow

### Creating a New Tool

```bash
# 1. Create a new Rust binary crate targeting WASM
cargo new --bin my-tool
cd my-tool

# 2. Add the SDK dependency
cargo add agentos-sdk

# 3. Set WASM target
echo '[build]\ntarget = "wasm32-wasip1"' >> .cargo/config.toml

# 4. Write your tool (src/main.rs)
# (use the #[tool(...)] macro)

# 5. Build
cargo build --release

# 6. Install into AgentOS
# The manifest is emitted to target/wasm32-wasip1/release/build/my-tool-*/out/tool-manifest.toml
agentctl tool install target/wasm32-wasip1/release/build/my-tool-*/out/tool-manifest.toml
```

### `agentos-sdk` CLI Helper (stretch goal)

```bash
# Scaffold a new tool project (like cargo new but for AgentOS tools)
agentos-cli sdk new my-tool --language rust

# Package everything into a distributable bundle (binary + manifest)
agentos-cli sdk package

# Install directly from the bundle
agentctl tool install my-tool-1.0.0.agpkg
```
